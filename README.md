# TF Submission: Justin Dingeman

1. The core concept students needed to understand
   - Students will need to understand how to collaborate with AI to drive application development from concept to implementation. They should know how to implement classes, helper methods and understand how to test AI written scripts. Students will also need to understand how to work with AI agents.
2. Where students are most likely to struggle
   - Students might struggle with understanding when to use Copilot "chat" vs. the custom built agent. The instructions don't make it clear what the difference is between them, and just says to switch back and forth. They may feel like it's just asking copilot the same questions over and over again since the chat already has the context from the project saved into the files. They may also struggle to keep up with the AI's implementations since it can work very quickly, and this assignment would actually take them much longer than 45 minutes if they were meant to be verifying in depth what the code should be doing.
3. Where AI was helpful vs. misleading
   - The AI can be helpful in coming up with various scripts but it'll definitely try to implement changes that are not part of the specifications despite being told to stick to the specifications. For example, in my session, it kept trying to add an additional class despite being told to only focus on the four classes.
4. One way they would guide a student without giving the answer
   - This one might be tough since this Tinker assignment really pushes towards quick development given its time constraint. Students might not really try to focus on it and just land on whatever the AI gives it. I feel that what I would likely do is to ask them to retrace their steps if they've gotten too far into using the AI for completing the project and have already lost sight of what the project is supposed to do.
